# UU MEMO 寫作與發布指南

歡迎來到您的專屬寫作空間。本指南將協助您高效地利用後台管理系統進行內容創作與影像管理。

---

## 🚀 核心寫作流程

### 1. 開啟編輯器
- 進入後台後，點擊左側導覽列的 **「文章編輯」**。
- 系統會自動帶入標準 Markdown 模板，包含必要的 `Frontmatter`（標題、描述、日期等）。
- 如果是編輯現有文章，請先在 **「文章管理」** 點擊編輯按鈕。

### 2. 設定文章元數據 (Metadata)
- **Slug**: 設定文章的網址路徑（純英文與連字號，如 `my-trip-to-japan`）。
- **發布狀態**: 使用編輯器上方的下拉選單設定：
    - 🟢 **Public (公開)**: 所有人可見。
    - 🟡 **Hidden (隱藏)**: 僅能透連結訪問，不顯示於首頁。
    - ⚪ **Draft (草稿)**: 僅供編輯使用，不會被建置。
- **精選文章**: 勾選後文章將顯示於首頁的橫幅區。

### 3. 影像工坊 (Image Workshop) ── 智慧插圖
這是最有效率的插圖方式：
1. **批次上傳**: 點擊「批次上傳照片」選擇您的照片。
2. **設定檔名與說明**:
    - **檔案名稱**: 系統會預設 `YYMM-slug-` 前綴，您可以微調檔名。
    - **圖片說明**: 在「圖片說明」框輸入這張照片的文字描述（將作為 Alt Text）。
3. **一鍵複製語法**: 
    - 點擊 **「一鍵複製語法」** 按鈕。
    - 系統會生成類似 `![您的說明](/images/2503/您的檔名.jpg)` 的內容並存入剪貼簿。
4. **貼上圖片**: 在 Markdown 編輯器中，將游標移至您想要放圖片的地點，直接貼上 (`Ctrl+V` / `Cmd+V`) 即可。

### 4. 預覽與儲存
- 編輯器的左側提供即時選單，您可以隨時切換至 **「文章管理」** 重新整理查看狀態。
- 雖然編輯器會自動同步狀態至內容中，但請確保最終點擊底部的 **「FINAL PUSH TO GITHUB」**。

### 5. 同步至 GitHub (發布)
- 點擊底部紅棕色的 **「FINAL PUSH TO GITHUB」** 按鈕。
- 確認視窗出現後點擊「確定」。
- 系統會將文章與圖片批次上傳至 GitHub，這會觸發網站自動建置，約 2-3 分鐘後即可看到更新。

---

## 💡 寫作小貼士
- **標題層級**: 建議文章內容從 `##` (H2) 開始，`#` (H1) 已由系統自動帶入文章標題。
- **YAML 語法**: 修改內容中的 `Frontmatter`（開頭的 `---` 區域）時請保持格式正確，建議優先使用介面選單。
- **本地備份**: 雖然有 GitHub 同步，重要的長文建議可以在本地 Markdown 編輯器留存一份。

---
